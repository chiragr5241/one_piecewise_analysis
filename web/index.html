About This Analysis

This visualization recreates the analysis from Not David's video on anime character networks. Characters are connected if they co-appear in at least one episode. Data scraped from the One Piece Wiki using the MediaWiki API.

Edge Weight — Friendship Score

For characters i and j, the friendship score is:

Fij = Nij / max(Ni, Nj)

where:

  • Nij = |{e : ai(e) = 1 ∧ aj(e) = 1}| — number of episodes both appear in
  • Ni = Σe ai(e) — total episodes character i appears in
  • max(Ni, Nj) — normalizes by the more prominent character's count

This gives Fij ∈ [0, 1]. Fij = 1 means every time the more active character appears, the other is always present too.

Degree Distribution

The degree ki of a character is the number of other characters they share at least one episode with. The histogram shows the probability P(k) that a randomly chosen character has degree k. One Piece's network is near-complete (mean degree ≈ 176), reflecting the interconnected nature of the story — most characters eventually share an episode with many others through flashbacks and ensemble scenes.

Network Construction Details

Graph Definition

The network is an undirected weighted graph G = (V, E, w) where:

  • V = set of all characters (|V| = 1,111)
  • E = {(i,j) : Nij > 0} — an edge exists if two characters ever co-appear
  • w(i,j) = Fij = Nij / max(Ni, Nj) — edge weight is the friendship score

Edge Threshold Filtering

The full graph has 97,930 edges — too dense to visualize. We apply a threshold τ and only display edges where Fij ≥ τ. Higher thresholds show only the strongest relationships; lower thresholds reveal weaker but still meaningful connections.

Eτ = {(i,j) ∈ E : Fij ≥ τ}

Layout Algorithm

The force-directed (CoSE) layout simulates a physical system: nodes repel each other (Coulomb's law) while edges act as springs (Hooke's law). The system converges to an equilibrium where connected nodes cluster together and disconnected groups separate naturally. This reveals the community structure without any prior knowledge of factions or arcs.

Character Appearance Raster

Each row is a character, each column is an episode. A colored cell means the character appeared in that episode. Characters are sorted by total episode count (descending). This is the same visualization from Not David's video, showing the characteristic triangular shape of serialized storytelling — new characters accumulate over time, but earlier characters keep reappearing.

Data collection: For each episode e, we scrape the "Characters in Order of Appearance" section from the One Piece Wiki. For character i, define the appearance vector:
ai(e) = 1 if character i appears in episode e, else 0
The raster is the binary matrix A ∈ {0,1}N×E where N = number of characters and E = number of episodes. The total appearances of character i are: Ni = Σe ai(e).

Community Detection (Louvain Algorithm)

The network was partitioned into communities using the Louvain method, which maximizes modularity. Each community roughly corresponds to a story arc or faction grouping.

Modularity

The Louvain algorithm maximizes the modularity Q, which measures how much more densely connected nodes within communities are compared to a random graph with the same degree sequence:

Q = (1/2m) Σij [Aij − (kikj)/(2m)] · δ(ci, cj)

where:

  • Aij = weighted adjacency (Fij if edge exists, 0 otherwise)
  • m = (1/2) Σij Aij — total edge weight
  • ki = Σj Aij — weighted degree (strength) of node i
  • δ(ci, cj) = 1 if nodes i and j are in the same community, else 0

Q ranges from −0.5 to 1. A Q of 0.563 indicates strong community structure — characters cluster tightly by story arc.

Centrality Measures — Mathematical Definitions

Degree Centrality

The fraction of all other nodes a character is connected to:

CD(i) = ki / (N − 1)

where ki is the number of characters that share at least one episode with character i, and N is the total character count.

Betweenness Centrality

The fraction of all shortest paths in the network that pass through a given character:

CB(i) = Σs≠i≠t [σst(i) / σst] / [(N−1)(N−2)/2]

where σst is the number of shortest paths from s to t, and σst(i) is the number of those paths passing through i. High betweenness = a bridge connecting otherwise distant groups (e.g., a character who appears in multiple story arcs).

Eigenvector Centrality

A recursive measure: a character is important if connected to other important characters. Defined as the leading eigenvector of the adjacency matrix:

A · x = λmax · x,    CE(i) = xi

where A is the weighted adjacency matrix and λmax is its largest eigenvalue. This captures "inner circle" importance — Straw Hats rank high because they are connected to each other AND to many other well-connected characters.

Clustering Coefficient

How much a character's neighbors are also neighbors of each other:

CC(i) = 2T(i) / [ki(ki − 1)]

where T(i) is the number of triangles through node i. Low clustering (0.056 average) reflects the hub-and-spoke nature of One Piece: the Straw Hats connect many characters who don't otherwise co-appear.

Multiplex Network — How It Works

Layer 1: Co-appearance (Jaccard + NPMI)

Blends two co-presence metrics to measure how much two characters appear together relative to chance:

Jaccardij = |Ei ∩ Ej| / |Ei ∪ Ej|
NPMIij = log2(P(i,j) / P(i)P(j)) / −log2(P(i,j))

NPMI (Normalized Pointwise Mutual Information) penalizes universal screen hogs better than raw co-appearance count.

Layer 2: Arc Overlap

Jaccard index on story arcs rather than episodes. Two characters sharing multiple key arcs is more meaningful than background co-appearances:

ArcOverlapij = |Ai ∩ Aj| / |Ai ∪ Aj|

Layer 3: Affiliation

Scores based on shared organizational membership: same crew = 1.0, allied orgs = 0.4, enemy orgs = −0.7. This is a signed layer.

Layer 4: Explicit Relationships

Extracted from wiki Relationships sections. Classified into family (+1.0), crew (+0.9), mentor (+0.8), ally (+0.6), rival (−0.2), enemy (−1.0). Also a signed layer.

Layer 5: Hierarchy

Organizational rank proximity within the same group: score = 1 / (1 + 0.3·|ranka − rankb|)

Layer 6: Conflict

Hostility edges derived from negative relationship scores and opposing faction membership. Higher score = stronger enmity.
